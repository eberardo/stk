stk_noga.molecular.molecules moduleÂ¶

Defines classes which describe molecules.

There are a couple of major classes defined here. The most important ones are StructUnit and MacroMolecule. Here is an overview of their role and interaction. This is followed by a step-by-step guide to macromolecular assembly.

StructUnit represents the monomers that make up macromolecules. These are commonly refered to as building blocks in the documentation. StructUnit holds information concerning only a single building block molecule. For example, the number of atoms and bonds a building block may have. It also has information about the functoinal groups present in the building block molecule (see FGInfo). The class also allows manipulation of the lone building block molecule, such as rotations and translations.

The StructUnit should be inherited as necessary. For example, StructUnit2 adds manipulations relavant to molecules with 2 functional groups. StructUnit3 adds manipulations relavant to molecules with 3 or more functional groups. If you have a monomer which needs specific information or manipulations, give it its own class.

The MacroMolecule class represents an assembled macromolecule. It requires at least 2 basic pieces of information: which monomers are used to assemble the macromolecule and what the topology/structure of the macromolecule is.

MacroMolecule.building_blocks holds a list of StructUnit instances. These represent the monomers which make up the macromolecule. Only one StructUnit instance per monomer type is held. So if 4 of one type of monomer and 2 of another type of monomer form a macromolecule, only 2 StructUnit instances are in MacroMolecule.building_blocks.

MacroMolecule.topology holds a Topology instance. This instance is responsible for assembling the macromolecule from the building blocks. The building should happen in MacroMolecule.__init__() via Topology.build(). The build() method places the assembled macromoleclue in MacroMolecule.mol as an rdkit molecule.

StructUnit labels atoms in the functional groups of the building blocks as either 'bonder' or 'del' (see its documentation). This tells the Topology instance which atoms form bonds and which are removed during assembly.

A more detailed description of macromolecular assembly.Â¶

This is a step-by-step guide of how macromolecular assembly is carried out and what the classes do.

First you create StructUnit instances of the building blocks which make up the macromolecule:

bb = StructUnit('/path/to/struct/file.mol2', 'amine')

The StructUnit instances are initialized using paths to molecular structure files. (Initializing a StructUnit automatically completes steps 1 to 4.)

  1. Place an rdkit instance of the molecule into StructUnit.mol, i.e.

    bb.mol  # <rdkit.Chem.rdchem.Mol at 0x7f961a8f1f80>

  2. Scan the path of the structure file for the name of a functional group. (Alternatively the name of a functional group can be supplied to the initializer).

    bb.func_group.name  # 'amine'

Which functional groups are recognized by stk?

The module fg_info defines the class FGInfo and a list of instances of this class called functional_groups. If you put an FGInfo instance into functional_groups, the functional group will be recognized.

  1. Place the FGInfo instance of the functional group into StructUnit.func_grp.

  2. Using FGInfo, tag atoms in the building block as either 'bonder' or 'del'. 'bonder' signifies that the atoms form a bond during macromolecular assembly, while 'del' means they are deleted.

  3. Give the StructUnit and Topology instances to the macromoleculeâs initializer.

    macro_mol = MacroMolecule([bb1, bb2], Topology())

    Normally, MacroMolecule and Topology will not be used directly. Instead, classes derived from these will be used. For example,

    macro_mol = Polymer([bb1, bb2], Linear("AB", [0, 0], 3))

  4. Run Topology.build() inside MacroMolecule.__init__().

  5. The details of Topology.build() will vary depending on the Topology class used. However, the basic structure is the same (steps 8 - 10).

  6. Combine the rdkit molecules in StructUnit.mol into a single rdkit molecule. Make sure that the building blocks are arranged in the shape of the macromolecule. All the manipulations available through StructUnit are useful here to make sure all building blocks are oriented correctly when forming the macromolecule.

  7. Create bonds between all the disjoined building block molecules. This is where the tagging done by StructUnit is needed.

  8. Delete all atoms tagged for deletion.

After all this you should have a rdkit instance of the macromolecule which should be placed into MacroMolecule.mol.

Extending stk: Adding new macromolecules.Â¶

To add new macromolecules create a new class which inherits MacroMolecule.

If youâre adding a new class of macromolecules, it quite likely you want to add a new Topology class. See the topologies.base for guidance on adding these. The topology class does the assembly of the macromolecule from the building blocks.

class stk_noga.molecular.molecules.MacroMolecule(building_blocks, topology, name='', note='', bb_conformers=None)[source]Â¶

Bases: stk_noga.molecular.molecules.Molecule

A representing assembled macromolecules.

Because of the computational cost associated with macromolecule assembly, instances of this class are cached. This means that providing the same arguments to the initializer will not build a different instance with the same attribute values. It will yield the original instance, retrieved from memory.

This class is not intended to be used directly but should be inherited by subclasses representing specific macromolecules. The Cage and Polymer classes are examples of this. Any information or methods that apply generally to all macromolecule should be defined within this class while specific non-general data should be included in the derived classes.

Note the equality operator == compares fitness. This means two different macromolecules compare equal if they happen to have the same fitness. The operator is not to be used to check if one macromolecule is the same structurally as another. To do this, use same().

set_orientation(start, end, conformer=-1)[source]Â¶

Rotates the molecule by a rotation from start to end.

Given two direction vectors, start and end, this method applies the rotation required transform start to end on the molecule. The rotation occurs about the centroid of the molecule.

For example, if the start and end vectors are 45 degrees apart, a 45 degree rotation will be applied to the molecule. The rotation will be along the appropriate direction.

The great thing about this method is that you as long as you can associate a geometric feature of the molecule with a vector, then the molecule can be rotated so that this vector is aligned with end. The defined vector can be virtually anything. This means that any geometric feature of the molecule can be easily aligned with any arbitrary axis.

Notes

The difference between this method and StructUnit._set_orientation2() is about which point the rotation occurs: centroid of the entire molecule versus centroid of the bonder atoms, respectively.

Parameters:
  • start (numpy.array) â A vector which is to be rotated so that it transforms to the end vector.
  • end (numpy.array) â This array holds the vector, onto which start is rotated.
  • conformer (int, optional) â The id of the conformer to use.

_set_orientation2(start, end, conformer)[source]Â¶

Rotates the molecule by a rotation from start to end.

Given two direction vectors, start and end, this method applies the rotation required transform start to end on the molecule. The rotation occurs about the centroid of the molecule.

For example, if the start and end vectors are 45 degrees apart, a 45 degree rotation will be applied to the molecule. The rotation will be along the appropriate direction.

The great thing about this method is that you as long as you can associate a gemotric feature of the molecule with a vector, then the molecule can be roatated so that this vector is aligned with end. The defined vector can be virtually anything. This means that any geometric feature of the molecule can be easily aligned with any arbitrary axis.

Notes

The difference between this method and set_orientation() is about which point the rotation occurs: centroid of bonder atoms versus centroid of entire molecule, respectively.

Returns the coefficients of the plane formed by bonder atoms.

A plane is defined by the scalar plane equation:

ax + by + cz = d.

This method returns the a, b, c and d coefficients of this equation for the plane formed by the bonder atoms. The coefficents a, b and c decribe the normal vector to the plane. The coefficent d is found by substituting these coefficients along with the x, y and z variables in the scalar equation and solving for d. The variables x, y and z are substituted by the coordinates of some point on the plane. For example, the position of one of the bonder atoms.
