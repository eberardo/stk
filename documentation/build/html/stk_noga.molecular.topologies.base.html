Extending stk: Adding new topologies.¶

Cages¶

To add a new cage topology a new class should be created, named after the topology. This class should inherit _CageTopology. This will give access to various methods which are necessary for dealing with any cage molecule. See the documenation of _CageTopology for more details.

The new class will only need to have five class attributes added:

  1. a list called vertices
  2. a list called edges
  3. n_windows, which holds the number of windows the cage topology has.
  4. n_window_types, which holds the number of different window types. For example, if n_window_types is 2, then the topology will have two kinds of windows, each with a different expected size. Windows of the same type are expected to be of the same size.

vertices holds instances of Vertex. Each instance represents a vertex of a cage and needs to be initialized with the coordinates of that vertex. Vertices of a cage are where building blocks of cages are placed.

edges holds instances of the Edge. Each instance represents an edge of a cage and needs to be initialized with two instances of Vertex. The Vertex instances should be held in vertices, as mentioned above. The two vertices are the ones which the edge connects. Linkers of cages are placed on edges. The edge instances automatically derive their positions from the vertices supplied during initialization.

The vertices need to be positioned such that the center of the topology is at the origin.

A class represting linear polymers.

repeating_unit¶

str – A string showing the repeating unit of the Polymer. For example, "AB" or "ABB". The building block with index 0 in MacroMolecule.building_blocks is labelled as "A" while index 1 as "B" and so on.

orientation¶

tuple of int – For each character in the repeating unit, a value of -1, 0 or 1 must be given in a list. It indicates the direction at which each monomer of the repeating unit is placed along the chain. 0 means that the direction is random.

n¶

int – The number of repeating units which are used to make the polymer.

ends¶

str – The string represents how the end groups of the polymer are treated. If 'h' the functional groups at the end of the polymer are converted into hydrogem atoms. If 'fg' they are kept as the original functional group.

class stk_noga.molecular.topologies.base.Topology[source]¶

Bases: object

Builds macromolecules.

More accurately, child classes of Topology take care of building macromolecules.

This class directly defines any operations and attributes that are needed by any topology during assembly. However, this class is not used directly. It is intended to be inherited from. All MacroMolecule.topology attributes hold an instance of a Topology child class. Child classes of Topology define operations specific to that one topology. For example, each child class must define a join_mols(), which creates bonds between the building blocks of a macromolecule. The way in which this is done will depend on what kind of macromolecules are being built. In addition, each child class must define methods which place the building blocks in approriate positions.

static determine_bond_type(macro_mol, atom1_id, atom2_id)[source]¶

Returns the bond order to be formed between the atoms.

Some atoms will need to have a double bond created between them. This is defined in the fg_info.double_bond_combs. If the atom ids provided as parameters belong to functional groups found in this list, the rdkit double bond type will be returned. If not, the rdkit single bond type will be returned. These types are needed when adding bonds to an rdkit molecule.

Parameters:
  • macro_mol (MacroMolecule) – The macromolecule being assembled.
  • atom1_id (int) – The id number of the first atom.
  • atom2_id (int) – The id number of the second atom.
Returns:

  • rdkit.Chem.rdchem.BondType.SINGLE – If the atoms don’t belong to functional groups which form a double bond.
  • rdkit.Chem.rdchem.BondType.DOUBLE – If the atoms belong to functional groups which form a double bond.
